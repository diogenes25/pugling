# Git Basics — Curriculum

**Level:** beginner · **Total:** 20 points · **Pass bar:** 16 / 20 (80%)

A short, hands-on introduction to everyday version control with Git. Two modules, worked at the
command line. Authored once by the `lehrplan-autor` skill; worked through unsupervised via the `lehrplan-lerner` skill.

## Modules

| # | Title | Points | Module pass bar |
|---|-------|--------|-----------------|
| 01 | The three states: working tree, staging area, repository | 10 | 70% (7/10) |
| 02 | Branching and merging | 10 | 70% (7/10) |

Each module has three exercises: a multiple-choice check, a hands-on code task (real git commands in
a throwaway repo), and a short "explain in your own words" question graded against a rubric.

## House rules

- Work the modules in order — Module 02 assumes Module 01 is passed.
- Attempt every exercise and write the answer into `ledger.md` **before** opening the answer key.
- For code exercises, actually run the commands — don't imagine the output.
- Grade honestly against the rubrics. The ledger is auditable.
- The course is not complete until overall points reach 16/20.
